# 공공직군 행정업무 슈퍼앱 PRD

## 1. 개요

공공직군 행정업무 슈퍼앱은 팀 단위 행정업무에서 반복적으로 발생하는 일정 공유, 엑셀 처리, 민원 대응, 행정 뉴스 모니터링을 하나의 업무 포털에서 처리하기 위한 내부용 웹 애플리케이션이다.

초기 버전은 실무자가 매일 사용하는 기능을 빠르게 제공하는 MVP를 목표로 하며, 이후 권한 관리, 감사 로그, 기관별 커스터마이징, 외부 시스템 연동을 확장한다.

## 2. 목표

- 팀원 휴가, 근무, 출장 일정을 한 화면에서 공유한다.
- 반복적인 엑셀 분리, 병합 업무를 웹에서 자동 처리한다.
- 민원 매뉴얼 기반으로 대응 방향과 응대 스크립트 초안을 생성한다.
- 공공 행정 관련 뉴스를 매일 아침 자동 수집해 업무 참고 자료로 제공한다.
- TypeScript, Vite, React, FastAPI, SQLite 기반의 단순하고 빠른 개발 구조를 유지한다.

## 3. 사용자

| 사용자 | 주요 목적 |
| --- | --- |
| 팀장 | 팀원 일정 파악, 업무 공백 확인, 주요 이슈 확인 |
| 팀원 | 본인 일정 등록, 타 팀원 일정 확인, 엑셀 업무 자동화 |
| 민원 담당자 | 민원 매뉴얼 기반 응대 스크립트 작성 |
| 행정 담당자 | 관련 뉴스 확인, 정책 및 행정 이슈 모니터링 |
| 시스템 운영자 | 데이터 백업, 실행 상태 점검, 오류 대응 |

## 4. 핵심 기능

### 4.1 팀원 스케쥴 관리

#### 기능 설명

팀원들의 휴가, 근무, 출장, 교육, 재택 등 일정을 캘린더 형태로 등록하고 공유한다.

#### 주요 요구사항

- 월간, 주간, 일간 캘린더 뷰를 제공한다.
- 일정 유형을 구분한다.
  - 휴가
  - 근무
  - 출장
  - 교육
  - 기타
- 일정 등록, 수정, 삭제를 지원한다.
- 일정에는 제목, 담당자, 유형, 시작일시, 종료일시, 메모를 포함한다.
- 팀원별 필터와 일정 유형별 필터를 제공한다.
- 오늘 일정과 이번 주 주요 일정을 대시보드에 요약 표시한다.

#### MVP 범위

- 단일 팀 기준 일정 관리
- SQLite 저장
- 캘린더 목록 조회 및 CRUD API

#### 향후 확장

- 부서별 권한
- 반복 일정
- 휴가 승인 워크플로우
- Outlook, Google Calendar 연동

### 4.2 엑셀 업무 자동화

#### 기능 설명

사용자가 업로드한 엑셀 파일을 특정 컬럼 기준으로 나누거나, 여러 엑셀 파일을 하나로 합친다.

#### 주요 요구사항

- `.xlsx`, `.xls`, `.csv` 파일 업로드를 지원한다.
- 엑셀 헤더를 분석해 컬럼 목록을 표시한다.
- 특정 컬럼 값 기준으로 파일을 분리한다.
- 여러 엑셀 파일을 하나로 병합한다.
- 결과 파일은 다운로드 가능한 압축 파일 또는 단일 엑셀 파일로 제공한다.
- 처리 결과 요약을 제공한다.
  - 원본 행 수
  - 생성 파일 수
  - 오류 행 수
  - 처리 시간

#### MVP 범위

- `.xlsx` 파일 업로드
- 단일 컬럼 기준 분리
- 동일 헤더 파일 병합
- 처리 결과 다운로드

#### 향후 확장

- 컬럼 매핑 병합
- 템플릿 기반 정제
- 개인정보 마스킹
- 처리 이력 관리

### 4.3 민원 대응 챗봇

#### 기능 설명

첨부된 민원 매뉴얼을 기반으로 민원 내용에 대한 대응 방향, 확인 사항, 응대 스크립트 초안을 작성한다.

#### 주요 요구사항

- 민원 매뉴얼 파일을 업로드하고 관리한다.
- 사용자는 민원 내용을 입력한다.
- 시스템은 관련 매뉴얼 내용을 검색한다.
- 검색된 근거를 바탕으로 응대 초안을 생성한다.
- 답변은 다음 구조를 가진다.
  - 민원 요약
  - 확인해야 할 사항
  - 대응 방향
  - 응대 스크립트
  - 참고 근거
- 생성 답변에는 최종 검토 필요 문구를 표시한다.

#### MVP 범위

- 매뉴얼 파일 업로드
- 텍스트 기반 검색
- 대응 스크립트 초안 생성 화면
- 답변 저장 없이 단회성 응답

#### 향후 확장

- 벡터 검색
- 기관 내부 LLM 또는 외부 LLM 연동
- 상담 이력 저장
- 민감정보 탐지 및 비식별화
- 답변 품질 평가

### 4.4 뉴스 기사 수집

#### 기능 설명

공공 행정, 지방행정, 복지, 안전, 인사, 예산 등 업무 관련 뉴스를 매일 아침 자동 수집한다.

#### 주요 요구사항

- 키워드 기반 뉴스 수집을 지원한다.
- 수집 주기는 매일 아침으로 설정한다.
- 수집된 기사는 제목, 언론사, 발행일, 링크, 요약, 키워드를 저장한다.
- 대시보드에서 최신 뉴스 목록을 제공한다.
- 키워드별 필터를 제공한다.
- 중복 기사는 저장하지 않는다.

#### MVP 범위

- 수동 실행 가능한 뉴스 수집 API
- 키워드 목록 기반 수집
- SQLite 저장
- 최신 기사 목록 조회

#### 향후 확장

- Windows 작업 스케줄러 또는 서버 Cron 연동
- 메일, 메신저 알림
- 기사 요약 자동화
- 부서별 관심 키워드 관리

## 5. 화면 요구사항

| 화면 | 주요 내용 |
| --- | --- |
| 대시보드 | 오늘 일정, 이번 주 일정, 최신 뉴스, 자주 쓰는 업무 자동화 |
| 일정 관리 | 캘린더, 일정 등록 모달, 팀원 필터 |
| 엑셀 자동화 | 파일 업로드, 컬럼 선택, 분리/병합 실행, 결과 다운로드 |
| 민원 챗봇 | 매뉴얼 업로드, 민원 입력, 응대 초안 출력 |
| 뉴스 | 기사 목록, 키워드 필터, 수집 실행 |
| 설정 | 팀원 관리, 뉴스 키워드 관리, 매뉴얼 관리 |

## 6. 데이터 요구사항

| 데이터 | 설명 |
| --- | --- |
| team_members | 팀원 기본 정보 |
| schedules | 일정 정보 |
| excel_jobs | 엑셀 처리 작업 이력 |
| complaint_manuals | 민원 매뉴얼 파일 정보 |
| news_articles | 수집된 뉴스 기사 |
| news_keywords | 뉴스 수집 키워드 |

## 7. 비기능 요구사항

- 초기 버전은 내부망 또는 로컬 실행을 우선한다.
- SQLite 파일은 주기적으로 백업 가능해야 한다.
- 업로드 파일은 처리 후 보관 정책을 명확히 한다.
- 민원 데이터에는 개인정보가 포함될 수 있으므로 로그 저장을 최소화한다.
- API 오류는 사용자에게 이해 가능한 메시지로 제공한다.
- 엑셀 처리 중 실패한 행과 원인을 확인할 수 있어야 한다.

## 8. MVP 완료 기준

- FE와 BE가 각각 실행된다.
- React 화면에서 주요 메뉴가 이동 가능하다.
- FastAPI에서 일정, 엑셀, 민원, 뉴스 관련 기본 API가 제공된다.
- SQLite에 기본 데이터가 저장된다.
- 엑셀 파일 분리 또는 병합 중 하나 이상이 실제 동작한다.
- 뉴스 수집은 수동 실행으로 동작한다.
- 운영자가 문서를 보고 로컬 실행을 재현할 수 있다.
